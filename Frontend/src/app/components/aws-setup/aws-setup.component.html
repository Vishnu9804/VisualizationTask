To allow this application to visualize your AWS infrastructure, you must create an IAM Role in your AWS account using an Identity Provider (OIDC).

  1. Go to the AWS IAM Console -> Roles -> Create Role.
  2. Select Web identity. Choose your Auth0 application as the Identity Provider.
  3. Attach a policy that grants ReadOnlyAccess (or custom describe permissions for EC2, VPC, RDS, S3).
  4. CRITICAL SECURITY STEP: Edit the Role's Trust Relationship to ensure only you can assume it. Copy and paste the policy below into the Trust Relationships tab:
{{ '{' }}
  "Version": "2012-10-17",
  "Statement": [
    {{ '{' }}
      "Effect": "Allow",
      "Principal": {{ '{' }}
        "Federated": "arn:aws:iam::YOUR_AWS_ACCOUNT:oidc-provider/YOUR_AUTH0_DOMAIN/"
      {{ '}' }},
      "Action": "sts:AssumeRoleWithWebIdentity",
      "Condition": {{ '{' }}
        "StringEquals": {{ '{' }}
          "YOUR_AUTH0_DOMAIN/:sub": "{{ auth0UserId }}"
        {{ '}' }}
      {{ '}' }}
    {{ '}' }}
  ]
{{ '}' }}
